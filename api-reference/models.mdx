## Overview

Get a complete list of available audio generation models with detailed specifications. Choose the optimal model based on quality requirements, processing speed, and bandwidth constraints.

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token with your API key: `Bearer sk-your-api-key-here`
</ParamField>

## Examples

<CodeGroup>

```bash cURL
curl -X GET "https://api.neospeech.io/v1/models/list" \
  -H "Authorization: Bearer sk-your-api-key-here"
```

```javascript JavaScript
const apiKey = process.env.NEOSPEECH_API_KEY;

async function listModels() {
  const response = await fetch('https://api.neospeech.io/v1/models/list', {
    method: 'GET',
    headers: {
      'Authorization': `Bearer ${apiKey}`
    }
  });

  if (!response.ok) {
    const error = await response.json();
    throw new Error(error.message);
  }

  const result = await response.json();
  return result.data;
}

// Get all models
const modelData = await listModels();
console.log(`Available models: ${modelData.total}`);

// Display model details
modelData.models.forEach(model => {
  console.log(`${model.name}: ${model.quality} quality at ${model.sample_rate}`);
});

// Get recommended models
const recommended = modelData.models.filter(m => m.recommended);
console.log('Recommended models:', recommended.map(m => m.id));
```

```python Python
import requests
import os

def list_models():
    response = requests.get(
        'https://api.neospeech.io/v1/models/list',
        headers={
            'Authorization': f'Bearer {os.getenv("NEOSPEECH_API_KEY")}'
        }
    )

    if response.status_code == 200:
        result = response.json()
        return result['data']
    else:
        error = response.json()
        raise Exception(error.get('message', 'Request failed'))

# Get all models
model_data = list_models()
print(f'Available models: {model_data["total"]}')

# Display model details
for model in model_data['models']:
    print(f'{model["name"]}: {model["quality"]} quality at {model["sample_rate"]}')

# Get recommended models
recommended = [m for m in model_data['models'] if m['recommended']]
print('Recommended models:', [m['id'] for m in recommended])
```

## Model Selection Helper

Choose the right model programmatically:

```javascript
function selectModel(requirements) {
  const { priority, useCase, latencyTarget } = requirements;

  // Quality priority
  if (priority === 'quality') {
    return 'aurora-4';
  }

  // Speed priority
  if (priority === 'speed' && latencyTarget < 1000) {
    return 'turbo-3';
  }

  // Use case based selection
  const useCaseMap = {
    'broadcasting': 'aurora-4',
    'podcast': 'aurora-3.5',
    'audiobook': 'aurora-3.5',
    'elearning': 'aurora-3.5',
    'chatbot': 'turbo-3',
    'ivr': 'turbo-3',
    'mobile': 'mini-2',
    'iot': 'mini-2'
  };

  return useCaseMap[useCase] || 'aurora-3.5';
}

// Examples
const model1 = selectModel({ priority: 'quality' });
// Returns: 'aurora-4'

const model2 = selectModel({ useCase: 'chatbot' });
// Returns: 'turbo-3'

const model3 = selectModel({ priority: 'speed', latencyTarget: 800 });
// Returns: 'turbo-3'
```

## Quality vs Speed Trade-offs

<AccordionGroup>
  <Accordion title="Premium Quality (Aurora 4)">
    **Pros:** Best audio fidelity, natural sound, suitable for professional use

    **Cons:** Higher latency (~2.8s), larger file sizes, more bandwidth required

    **When to use:** Quality is critical, latency is acceptable, professional output required
  </Accordion>

  <Accordion title="Balanced (Aurora 3.5)">
    **Pros:** Excellent quality-to-speed ratio, versatile, recommended for most uses

    **Cons:** Not the fastest or highest quality, may not fit extreme requirements

    **When to use:** Most general-purpose applications, good compromise
  </Accordion>

  <Accordion title="Fast Processing (Turbo 3)">
    **Pros:** Low latency (~0.9s), quick response, real-time capable

    **Cons:** Lower audio quality, smaller sample rate, less natural

    **When to use:** Speed is critical, interactive applications, real-time requirements
  </Accordion>

  <Accordion title="Lightweight (Mini 2)">
    **Pros:** Small file sizes, efficient, low resource usage

    **Cons:** Basic quality, limited use cases, compromised audio fidelity

    **When to use:** Resource constraints, low bandwidth, mobile/IoT devices
  </Accordion>
</AccordionGroup>
